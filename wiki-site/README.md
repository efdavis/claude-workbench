# wiki-site

A [Docusaurus](https://docusaurus.io/) shell that renders a personal Markdown
wiki as a fast, searchable static site. Vendor-neutral — it ships with no
content of its own; you point it at a directory of Markdown articles.

Pairs with the `/wiki-store`, `/wiki-ask`, `/wiki-compile` skills in this repo:
those skills maintain the article files, this renders them. Both read the same
`CLAUDE_WIKI_DIR` env var.

## What it gives you

- **Obsidian-compatible `[[wikilinks]]`** — `[[article]]`, `[[article|alias]]`,
  `[[article#heading]]` rewrite to working page links at build time, so the same
  files render in an Obsidian vault and here.
- **Backlinks panel** under every article (which other articles link here),
  generated by `scripts/generate-backlinks.mjs`.
- **Local full-text search** (no external service).
- **Auto-generated sidebar** from your articles dir — no slugs hardcoded.
- **`static/infographics/`** for standalone single-file HTML (served verbatim).
- A dinosaur "national park poster" landing page (CC0 art); entry/crossing
  counts are read live from the generated backlinks data.

## Quick start

```bash
npm install
npm start          # → http://localhost:3200  (renders the bundled example-wiki/)
```

Point it at your own wiki:

```bash
export CLAUDE_WIKI_DIR=~/wiki     # articles must live in $CLAUDE_WIKI_DIR/articles/
npm start
```

Requires Node >= 20.

## Configuration (all via env vars)

| Var | Default | Purpose |
|-----|---------|---------|
| `CLAUDE_WIKI_DIR` | `./example-wiki` | Wiki **root**. Articles are read from `<root>/articles/*.md`. `~` is expanded. Shared with the `/wiki-*` skills. |
| `WIKI_TITLE` | `My Wiki` | Site + navbar title |
| `WIKI_TAGLINE` | `Notes, rendered.` | Hero tagline |
| `WIKI_AUTHOR` | _(empty)_ | Footer copyright name |
| `WIKI_URL` | `https://your-site.example.com` | Production URL (deploy only) |
| `WIKI_BASE_URL` | `/` | Base path, e.g. `/wiki-site/` for a GitHub Pages project site |
| `WIKI_GH_ORG` / `WIKI_GH_REPO` | `your-org` / `wiki-site` | `npm run deploy` target |

## Expected wiki layout

```
$CLAUDE_WIKI_DIR/
└── articles/
    ├── INDEX.md        # frontmatter `slug: /` → becomes the /wiki/ landing page
    └── *.md            # one concept per file, kebab-case names
```

Article frontmatter, link, and authoring conventions are in
`../commands/references/wiki-conventions.md`. The bundled `example-wiki/`
demonstrates the minimum: an `INDEX.md` plus two cross-linked articles.

## Set it up on another computer

1. Clone this repo, `cd wiki-site`, `npm install`.
2. Get your wiki onto the machine (it is intentionally NOT bundled here — it is
   your content). Put it anywhere.
3. `export CLAUDE_WIKI_DIR=/path/to/your/wiki` (add to your shell profile to
   persist).
4. `npm start`.

Nothing about the site assumes a specific machine or a sibling directory — the
only coupling is the `CLAUDE_WIKI_DIR` path.

## Build & deploy

```bash
npm run build      # static output in ./build
npm run serve      # preview the build locally on :3200
npm run deploy     # GitHub Pages (set WIKI_GH_ORG/WIKI_GH_REPO/WIKI_BASE_URL first)
```

## Layout

```
docusaurus.config.ts   # resolves CLAUDE_WIKI_DIR; [[wikilink]] preprocessor; search
sidebars.ts            # single autogenerated sidebar
scripts/
  generate-backlinks.mjs   # runs on prestart/prebuild; writes static/backlinks.json
src/
  pages/index.tsx          # poster landing page
  theme/DocItem/Footer/    # injects the backlinks panel
  css/custom.css           # national-park palette
static/
  infographics/            # standalone HTML, served verbatim
  img/                     # logo, favicon, CC0 dino art
example-wiki/articles/     # bundled demo content (override with CLAUDE_WIKI_DIR)
```
