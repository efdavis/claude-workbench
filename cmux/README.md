# cmux

My [cmux](https://cmux.com) terminal UI and settings. cmux is a Ghostty-based
terminal; its appearance comes from a Ghostty config plus a cmux app-settings
file. This folder carries the two files I hand-edit. The color theme is managed
by a cmux CLI command (documented below), not by a tracked file.

## Files

| File | Symlink target | What it sets |
|------|----------------|--------------|
| `ghostty-config` | `~/.config/ghostty/config` | Font size, foreground brightening. cmux reads this for terminal appearance. |
| `cmux.json` | `~/.config/cmux/cmux.json` | Dark chrome (force dark mode), navy sidebar tint. |

The theme (`TokyoNight Storm`) is stored separately in
`~/Library/Application Support/com.cmuxterm.app/config.ghostty`, written by the
`cmux themes` command. That file is regenerated by cmux, so it is not tracked
here. Set it with the command below.

## Install

Back up anything you already have, then symlink:

```bash
# Back up existing files first
[ -f ~/.config/ghostty/config ] && cp ~/.config/ghostty/config ~/.config/ghostty/config.bak
[ -f ~/.config/cmux/cmux.json ] && cp ~/.config/cmux/cmux.json ~/.config/cmux/cmux.json.bak-$(date +%Y%m%d-%H%M%S)

# Symlink the tracked files
mkdir -p ~/.config/ghostty ~/.config/cmux
ln -sf ~/Projects/claude-workbench/cmux/ghostty-config ~/.config/ghostty/config
ln -sf ~/Projects/claude-workbench/cmux/cmux.json ~/.config/cmux/cmux.json

# Set the theme (writes both light and dark slots — see gotcha below)
cmux themes set "TokyoNight Storm"

# Apply everything live, no app restart
cmux reload-config
```

## What this gives you

* **Theme**: TokyoNight Storm (bg `#24283b`, fg `#c0caf5`, red `#f7768e`).
* **Foreground**: brightened to `#d6ddfa` (a touch above the theme's `#c0caf5`, not full white).
* **Font**: cmux's bundled JetBrains Mono default at size 17. `font-family` is
  left unset on purpose so the bundled font is used; Nerd Font glyphs still
  render via cmux's symbol fallback. Swap with `font-family = Maple Mono NF`.
* **Chrome**: forced dark (`app.appearance: "dark"`) so the sidebar/tabs stay
  dark even when the OS is in light mode.
* **Sidebar**: tinted navy with the terminal background instead of the default
  black tint.

## Gotchas

* **Theme must be set for BOTH light and dark slots.** `cmux themes set --dark X`
  only applies when the OS resolves to dark. With the OS in light mode a
  dark-only theme silently falls back to cmux's default (white fg on `#222333`).
  The bare `cmux themes set "TokyoNight Storm"` writes both slots
  (`theme = light:...,dark:...`) and applies unconditionally. ✅
* `app.appearance` controls the chrome (sidebar/tabs); the `theme` setting
  controls the terminal surface. They are separate — set both for a fully dark UI.
* Validate the JSONC before reloading: `cmux config check`.
* See every available config path: `cmux config paths`.
* Browse themes: `cmux themes list`. Theme files ship at
  `/Applications/cmux.app/Contents/Resources/ghostty/themes/`.
